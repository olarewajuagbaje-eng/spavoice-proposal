THE PROBLEM WE ARE SOLVING

Medical Spas Are Losing
Money on Every Missed Call

A medical spa is a high-value business. A single Botox appointment is $300-$600. A single missed call could be a $400 booking that went to the competitor next door. And most spas are missing calls every single day.

📞

Calls Go Unanswered

Staff are busy with clients in the room. The phone rings and rings. The caller hangs up and calls someone else. The spa lost $400 in 30 seconds.

🌙

No After-Hours Booking

Most people research and book outside business hours. A spa with no after-hours system loses every evening and weekend booking opportunity.

🔁

Staff Doing Repetitive Work

Answering the same questions, checking the same calendar, sending the same reminder texts. Highly paid staff doing work a system should handle.

❌

No-Shows Kill Revenue

A no-show is a blocked slot that could have been filled. Without automated reminders, no-show rates at medical spas average 15-20%.

🧩

Too Many Disconnected Tools

The spa uses one tool for bookings, another for client records, another for SMS. Nothing talks to each other. Staff bridge the gaps manually.

💸

Existing Solutions Are Expensive

Enterprise booking software costs thousands per month and requires dedicated IT staff. Small spas cannot afford or manage it.

HOW IT WORKS — PLAIN ENGLISH

What Happens When a
Patient Calls the Spa

This is the complete journey of a single phone call through the system. No technical knowledge needed to understand this — read it like a story.

1

The Patient Calls

A patient calls the spa's phone number — the same number they have always called. Nothing has changed on their end. But instead of a human picking up or voicemail taking over, an AI voice answers within two seconds. It sounds natural, warm, and professional. It knows the spa's name, services, and hours.

↓
2

The AI Understands What They Want

The patient says what they need — "I'd like to book a HydraFacial" or "I need to reschedule my appointment." The AI understands natural speech, not just button presses. It asks the right follow-up questions — what date, what time, any preferences — exactly like a trained receptionist would.

↓
3

It Checks Real Availability

The AI does not guess or make up times. It connects in real time to the spa's booking system — Vagaro, Mangomint, or Boulevard — reads what is actually available, and offers those specific slots to the patient. If Tuesday at 2pm is booked, the AI knows that and offers Wednesday instead.

↓
4

It Books Directly Into Their System

Once the patient confirms a time, the AI writes the appointment directly into the spa's existing booking system. The spa owner refreshes their calendar and the appointment is already there — no human touched it. The patient's name, service, and contact details are all captured.

↓
5

The Reminder Goes Out Automatically

The day before the appointment, the system automatically sends the patient a text message reminder. They can reply CONFIRM or text back to cancel. If they cancel, the slot opens back up in the spa's calendar immediately. No staff involvement at any point in this process.

↓
6

The Spa Owner Sees Everything in One Place

The spa owner logs into their simple dashboard. They see how many calls were handled, how many bookings were made, how many reminders were sent. They can update their services, prices, and hours themselves. They never need to call a developer or touch any technical system.

THE TOOLS WE ARE USING AND WHY

Every Tool Has One Job.
None of Them Overlap.

We chose each tool because it is the best at its specific job. Here is why we picked each one and what it does in plain language.

🌿
Lovable
THE INTERFACE BUILDER

Lovable builds beautiful, functional web applications rapidly. This is what the spa owners see when they log in to manage their AI receptionist. It also powers Rita's operator dashboard where she manages all her spa clients.

Why Lovable and not something else: It builds production-grade interfaces that look custom-designed without months of traditional development. The spa owner dashboard needs to be simple and beautiful — Lovable delivers exactly that.
🔐
Supabase
THE DATABASE AND SECURITY LAYER

Supabase stores everything — every spa's data, credentials, knowledge bases, and configurations. It handles login security, makes sure each spa can only see their own data, stores the AI's knowledge in a searchable format, and provides real-time updates across the entire system.

Why Supabase: It is the most secure, scalable, and developer-friendly database available. It handles multi-tenant isolation (keeping spas separated) out of the box, stores vector embeddings for the knowledge base, and encrypts sensitive data automatically.
⚙️
n8n
THE AUTOMATION ENGINE

n8n is the invisible engine that makes everything work together. It connects Vapi to the right CRM, automatically creates new agents when spas sign up, sends reminder messages, logs call data, and handles every automated process in the system.

Why n8n: It can connect to any API, has visual workflow building, runs reliably 24/7, and can handle complex routing logic without custom code for every step. When a CRM API changes, we update one n8n workflow — not the entire system.
📞
Vapi
THE AI VOICE PLATFORM

Vapi powers the actual phone conversations. Each spa gets their own Vapi agent — created automatically when they sign up. The agent knows only that spa's information and handles calls using natural AI conversation. Vapi has a full API so agent creation and configuration is 100% automated.

Why Vapi: Best-in-class voice AI with the cleanest API for automation. One Vapi account, unlimited agents, each fully isolated. The API-first design means no manual work per new spa client — n8n creates and configures everything automatically.
📱
Twilio
THE PHONE NUMBER PROVIDER

Twilio provides and manages all the phone numbers. Each spa gets their own dedicated number. SMS reminders go out through Twilio. At demo scale, numbers come through Vapi directly. As Rita scales past 20 clients, numbers move to a Twilio account Rita owns directly — giving full control over pricing and portability.

Why Twilio: Industry standard, most reliable, cheapest at scale (~$1.15/number/month), and fully programmable via API. If we ever need to switch voice providers, the numbers stay — no disruption to any spa's clients.
🔌
CRM Adapters
THE CUSTOM CONNECTORS

Each supported CRM gets its own dedicated connector built in n8n. The Vagaro connector only ever talks to Vagaro. The Mangomint connector only ever talks to Mangomint. They never mix. Supabase tells n8n which connector to use for each call based on which CRM that specific spa uses.

Why separate connectors: Isolation means one CRM changing their API never breaks another spa's bookings. Each connector can be updated, tested, and maintained independently. This is the correct engineering approach for multi-CRM support.

THE KNOWLEDGE BASE

How the AI Knows Everything
About Each Specific Spa

The AI does not come pre-loaded with information about every spa. Each spa teaches their AI about themselves. We made this as simple as possible — the spa owner can use their existing website, upload a document, or just type the information directly. They choose whichever is easiest for them.

🌐

Connect Their Website

The spa pastes their website URL. The system automatically reads all their pages — services, prices, about page, FAQ, team bios — and extracts the relevant information into the knowledge base. Takes 2 minutes.

e.g. https://glowaeshtetics.com
📄

Upload a Document

The spa uploads a PDF, Word doc, or text file containing their service menu, pricing list, FAQ document, or any other information they want the AI to know. The system processes and stores it automatically.

e.g. services-menu-2026.pdf
✏️

Type It Directly

Simple form fields. Service name, price, duration, description. Business hours. Location. Staff names. Common questions and answers. The spa fills in what they know and saves. The AI knows it immediately.

Service: Botox · $350 · 45 min

How the knowledge base works technically (simplified): When a caller asks a question — "Do you offer laser hair removal?" — the system searches the spa's knowledge base for the most relevant answer and gives it to the AI in real time. This is called RAG (Retrieval Augmented Generation). The AI does not guess — it finds the answer from the spa's actual content. If the information is not in the knowledge base, the AI politely says it does not have that information and offers to have someone follow up.

SECURITY AND DATA PROTECTION

Every Spa's Data Is
Completely Isolated.

Medical spas deal with clients' personal information, medical history, and payment details. Security is not optional — it is the foundation of trust. Here is how we protect every piece of data in the system.

🔒

Row-Level Security

Supabase enforces row-level security on every database table. This means Spa A's login credentials physically cannot read Spa B's data — the database itself blocks it, not just the application logic. Even if someone found a bug in the UI, the database would reject any cross-tenant data access.

🔑

Encrypted Credentials

When a spa connects their CRM, their API key is encrypted before it is ever stored. n8n never holds credentials — it requests them from Supabase fresh for each call, uses them once, and discards them. No credentials are ever logged or stored in workflow history.

🏠

Tenant Isolation

Each spa is a completely separate tenant in the system. Their knowledge base, their call history, their client data, their configuration — all stored in isolated sections of the database tagged only to their unique ID. No shared data pools between spas.

🛡️

Authentication via Supabase Auth

Every user login — spa owners and Rita — goes through Supabase's built-in authentication system. It handles password hashing, session management, and JWT tokens. No custom authentication code that could have vulnerabilities.

📋

Call Transcript Privacy

Call transcripts contain patient names and medical service details. Each transcript is stored encrypted in the relevant spa's isolated database partition. Only that spa's authenticated users can access their transcripts. Rita can see call volumes but not content without explicit access.

🔄

Automated Backups

Supabase runs automated daily backups of all data. If anything goes wrong, the entire system can be restored to a recent state. Spa owners never lose their configuration, knowledge base, or booking history.
